Creating a New Landing Page

  1. New Landing pages are to be created based on the requirement sheet received from the client. Requirement sheet contains all the details pertaining to Meta tags, creativity images, Master Key, text, template and many more details. Requirement sheet contains almost every detail required to complete a LP. Landing pages are created using a Campaign Automation tool – Eloqua.
    Link to FCA Marketing Eloqua-https://login.eloqua.com/auth/saml2/autologin?LoginPrefix=FIAM&Idp=61b8Ca75-060e-4392-9523-76e953661a77
    Link to FCA services Eloqua -https://login.eloqua.com
    Note: Both platforms/links have separate accesses and according to requirement we use respective Eloqua link.


  2. On successfully signing in to Eloqua tool we get to the above screen. On clicking the Edit icon following dropdown options are visible:
    1. Emails
    2. Landing pages
    3. Forms
    4. Components
    5. Email Setup


  3. On selecting Landing page Option (other Options explained later in this file) opens LP creation page with 4 menu cards and a table below showing recently accessed Eloqua LP’s. The 4 cards are-


    1. Create a Landing page: This option creates a blank Landing Page allowing to create a page from scratch. This approach is very rarely (in fact never) used for Creating Landing Page.
    2. Upload a Landing Page: This option allows a Landing page to be uploaded directly to Eloqua, which means that the page can be developed in HTML format outside Eloqua and then uploaded in HTML or template format. This type of page creation is used sometimes but still not the way landing pages are created.
    3. Manage Templates: This option allows reusing saved templates i.e. existing pages which can be used again instead of creating a page from scratch.
    4. Reporting Dashboard: This option redirects users to Oracle BI platform showing analytics data.(Specific access is needed)

What is the right way of Developing a LANDING PAGE???

Based on the country, brand etc. a previously developed Landing Page is copied and used as template for creating new LP. Generally we receive a template for the respective country and brand from the market. To make copy of the provided template in Eloqua please follow the below steps: -

  1. Browse the provided URL from the requirement sheet or request details for e.g. http://dpromo.jeep.com/de_Renegade and view source code of the page. At the bottom of the page you will see the Landing Page ID = 19482 (as seen in the screenshot for example) and copy the ID. Every page has a unique id.


  2. Access Eloqua tool and open any existing landing page(either from recently accessed list or from landing page menu under Edit)


  3. Notice that the browser url ends with id=****, just copy paste the page id copied from the source code of the page to be copied/cloned and browser the url.
    Notice that the page refreshes and the required template to be copied/cloned can be seen in Eloqua.


  4. Copy the Asset Name (LP Name) as shown in the screen shot to the right by double clicking on it.


  5. Now go back to the LP Page(by clicking Landing page from Edit menu) Search for this asset name in the search bar(see the screenshot).



    Notice the LP asset listed in the All files section and right click it to further copy the asset. Once copied a clone gets created(ends with “:copy”) in the same folder location


  6. Further right click on the clone asset and choose Move to Folder menu to move the cloned asset in the correct folder location.
  7. In the correct folder click Move to complete the file movement.


    8. Note: Please follow the below nomenclature while creating or moving new LP’s- Production: All>EMEA>Country>Brand>Year>Month>model Certification: All>Test>Mumbai_OP>Country>Brand>Year>Month>Model

  8. Once the LP is cloned it is ready for arranging further changes
  9. Notice that the asset is in the draft status (marked by number 1 in red color on the top left of the Eloqua screen). This suggests that you have not selected the micro site for the page.



  10. Select the relevant microsite(according to brand and county, do not consider mobile microsites) from the dropdown on top right for e.g. JOG-Desktop and click Save on top right of the page. Once saved the number will disappear.
  11. Page is ready for further work and you can follow below steps-
    1. Verify correctness of provided MK pdf’s i.e. is offer id, Campaign name, Campaign id & Process are same throughout all files, Campaign start date should be either before or same as golive/due date and end date is greater than golive/due date.
    2. Update Eloqua tags following below steps in Eloqua editor

  12. Update Cloudfront script, Base url, adobe script using the existing P2C macro file
  13. Also verify formname , form id, cookie id, Facebook id, linkedin id and privacy values using the same macro file
  14. Follow QC checklist ensuring relevant scripts are available and perform form testing using relevant values for each country. Also refer to new learning document for latest updates.
  15. Further proceed to perform UI and creativity changes/updates as per the shared brief.
  16. Once an LP is approved and push live is requested , arrange push live by updating the vanity text in the vanity box in top left (check screenshot in the right) and click save.
  17. Further verify it by browsing the final url and confirm push live.For e.g. http://dpromo.fiat.com/BENL_SummerDeals

Image/Asset Uploading

  1. Every Landing Page has many different images/assets. Inorder to use images, they are to be uploaded on Eloqua first. Uploading is done by browsing and selecting image from the system. It is important that the image is uploaded in respective folders for better storage and hassle free search. Folders are created based on certain important criteria such as: country, brand, model, etc
  2. Inorder to access the asset storage section go to the Edit icon and further click on Components>Images/File storage. Images option allows to upload images & File storage is used for uploading JS scripts, pdf files etc. Follow similar structure (listed below) for uploading files so that assets can be searched easily.
    Production: All>EMEA>Country>Brand>Year>Month>model
    Certification: All>Test> Country>Brand>Year>Month>model
