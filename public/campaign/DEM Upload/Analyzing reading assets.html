Analyzing (Reading) assets

In-order for a developer to arrange a DEM, a developer is provided with a requirement sheet, assets(html/images) to start with. Requirement sheet is very important as it provides the basic parameters for creating a DEM in Eloqua. So go through it carefully and ask the reporter if any parameter is missing or requirement file/html asset is missing.

Steps for analyzing the Requirement sheet:

Check for the below points in the requirement sheet before uploading the DEM

  1. Campaign name and subject line.
  2. Link to HTML file is provided or not(or atleast separately shared in the ticket).
  3. Campaign name should exactly match with ticket JIRA title.
  4. Check If the campaign name provided is used previously. If used, then make it unique by giving suffix ex. 1,A etc. after the actual campaign name. If not used, then proceed with same
  5. Remaining fields like country, brand, No. of assets to be uploaded etc

Please confirm with reporter for any doubts/queries before proceeding.
