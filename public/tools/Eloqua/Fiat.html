1. Introduction

The document has the purpose to illustrate the procedure in order to modify Fiat forms.

1.1 Assumption
  1. LPID is the unique ID of the Landing Page
  2. Each LP contains the form for a specific couple brand/market
    • For different layouts, there will be different landing pages (e.g. Standard, Promo and Fleet/Business forms)
  3. One LP is multiCTA/forms
  4. FIAT Prod Landing Pages use the Eloqua form “Portal_FIAT_Prod” associated with the “LeadEvent” Custom Object (except for IT “Portal_Italy_FIAT_Prod”)
  5. FIAT Test Landing Pages use the Eloqua form “TestFormAbarthAEM” associated with the “TestLeadEvent” Custom Object

1.2 Fiat Eloqua folder structure

Fiat Landing Pages are under All > FIAT path.

The main folder contains other three subfolders with the corresponding pages:

  1. TEST (Test LPID)
  2. TEST FORM (Certification LPID)
  3. LP FORM (Production LPID)

    4. Under these three folders there is a folder for each market that contains the related LPs:

    If not present, duplicate the Production LP into the TEST folder at the corresponding path in order to apply the changes needed. If already present, copy always the content of the production LP into the test one to start to work from the most updated version available in production.

    When you create a new Landing Page you have to set the Microsite.

    For Abarth it must be Fiat – Desktop.

    When done, report them in the corresponding PROD Landing Page.

    Remember that after modifying a Landing Page is always necessary a Fiat AEM portal refresh.

    In some cases, when the changes affect:

  4. Orchestrator values
  5. CCDB values
  6. adding and removing fields operation

they must be validated by the ICT team through the internal process on the AEM Test environment, before to be reported in AEM Prod.

This kind of changes must pass the ICT test before to be published in Production.


1.3 Landing Page Permissions

In order to integrate the Fiat Landing page into AEM Portal it is necessary to give it the correct permissions. Follow this steps:

  1. Open Landing page from Eloqua platform
  2. In the right corner click on Actions button
  3. Click on Permissions
  4. Add group CloudConnector and give it View permission;
  5. Save and save Landing page;

2. LPID Identification

The LPID is the unique ID of the Landing Page.

This chapter allows you to identify and locate the correct LPID of the form.

2.1 LPID browser research

To found quickly the LPID of the form to analyze, open the source code of the form in a browser (use a common developing tool).

The LPID, in the HTML code, is identified with a specific (allows you to locate it faster):

  1. tag:
  2. name: name="LPID"

    3. <input type="hidden" id="field62" name="LPID" value="10944"> (e.g. Fiat UK TD)

    2.2 LPID Eloqua research

    To found the corresponding Landing Page in Eloqua, having the LPID, go to any LP and change the “id” parameter directly in query string:

    https://secure.p06.Eloqua.com/Main.aspx#landing_pages&id={LPID}

    (e.g. for Fiat UK TD: https://secure.p06.Eloqua.com/Main.aspx#landing_pages&id=10944)

4. JS Management

This chapter allows you to modify the correct JavaScript files included in the Landing Page that you are analyzing.

4.1       Core JS structure

 

The files lq_standard_forms.js and lq_promotion_forms.js contain the core functionalities and are linked in the corresponding landing pages.

These files are a union of other JavaScript files and built through Fiat.Forms.UniqueFile.Generator.exe script.

This solution was adopted in order to separate basic functionalities and use them cross sites.

On the other hand, in the page the number of JS files included was reduced to improve performances and SEO ranking.

This files are stored on the Akamai server under https://staticpromo.fiat.com domain.

4.1.1        lq_standard_forms.js

 

The lq_standard_forms.js file is included in both Standard and Fleet/Business Landing Pages.

The lq_standard_forms.js, in the HTML code, is identified with by a specific (allows you to locate it faster):

<script type="text/javascript" src="https://staticpromo.fiat.com/FIAT/Portal/Core/lq_standard_forms.js"></script>

This is composed by the following JS files:

  • lq_init_form.js
  • digitalDataTracking.js
  • lq_privacy.js
  • lq_utility_function.js
  • lq_configuration.js
  • lq_submit.js
  • lq_dealerLocator.js

  • lq_preload_standard.js
4.1.2        lq_promotion_forms.js

The lq_promotion_forms.js, in the HTML code, is identified with by a specific (allows you to locate it faster):

4.1.4        Build Core JS

 

When you need to apply changes to the JS files you have to modify always the inner JavaScript files and regenerate the Core JS through the Fiat.Forms.UniqueFile.Generator.

In order to build Core JS, in the configuration file under Fiat.Forms.UniqueFile.Generator folder, set the following key:

  • InputPathName: the local input GIT path
  • OutputPathName: the local path which you want to create new file
  • formType: value must be
    • "standard" (standard forms)
    • "promotion" (promotion forms)
    • "fleet" (fleet form market IT)
  • outputFileName: value must be
    • “lq_standard_forms.js” for standard formTytpe
    • “lq_promotion_forms.js” for promotion formType
    • “lq_fleet_forms_IT.js” for fleet formType

Run console application to re-generate the Core JS files.

4.2       Market JS structure

 

The file lq_message_configuration_MARKET.js contains the messages and the configurations for the market and is linked in the corresponding landing pages.

The Market JS file is a union of other JavaScript files and built through Fiat.Forms.UniqueFile.Generator.exe script.

This solution was adopted in order to separate basic functionalities and use them cross sites.

On the other hand, in the page the number of JS files included was reduced to improve performances and SEO ranking.

This files are stored on the Akamai server under https://staticpromo.fiat.com domain.

4.2.1        lq_message_configuration_MARKET.js

 

The same lq_message_configuration_MARKET.js file is included in Standard, Promo and Fleet/Business Landing Pages for the specific market.

This is composed by the following JS files:

  • lq_form-message_MARKET.js
  • lq_form-configuration_MARKET.js

The lq_message_configuration_MARKET.js, in the HTML code, is identified with by a specific (allows you to locate it faster):

<script type="text/javascript" src=" https://staticpromo.fiat.com/FIAT/Portal/UK/lq_message_configuration_UK.js"></script>

(e.g. UK Market)

The lq_message_configuration_UK.js is obtained concatenating:

  • lq_form-message_UK.js
  • lq_form-configuration_UK.js
4.2.2        Build Market JS

 

When you need to apply changes to the JS files you have to modify always the inner JavaScript files and regenerate the Market JS through the Fiat.Forms.UniqueFile.Generator.

In order to build Market JS for messages and configuration, in the configuration file under Fiat.Forms.UniqueFile.Generator folder, set the following key:

  • InputPathName: the local input GIT path
  • OutputPathName: the local path in which you want to create new file
  • formType: value must be "message_MARKET"
  • outputFileName: value must be “lq_message_configuration_MARKET.js”

Run Console application to re-generate the Message JS files.

5. Label Configuration

This chapter allows you to modify correctly the message labels of your landing page.

 

5.1       Label replacement

 

If required to change a specific label, it is necessary to replace it in the specific Market JS file.

In particular, the labels are stored in the lq_form-message_MARKET.js subfile.

 

In this file there are several JS variables that contains JSONs with key and values of the specific labels.

The replacement is realized in the Core JS files through specific JavaScript functions.

The message, in the HTML code, is identified with a specific (allows you to locate it faster):

  • name="…"

        In the correspondin JSON you will find particular variables (for market/language):

var message =

  {

         uk:

                 {

                         en:

                                {

                                 …

                                "placeholder_nome": "Name*",

                                 …

                                }

                 }

  }                                         (e.g.lq_message_configuration_UK.js)

 

The text containd in the JSON value will replace the label of the <input> in the HTML code that has the same name=”” value of the JSON key after the “placeholder_” prefix.

 

<input type="text" class="form-control required" name="nome">             (e.g. Standard UK form)

 

The variables to change are in general:

var message = {} (general messages)

var select = {} (select options)

var tabName = {} (tab names)

var dealerMessage = {} (dealer page messages)

var stepMessage = {} (step messages)

var formLabel = {} (form labels like e.g. piva, submit button, fleet messages)

var privacyMessage = {} (privacy messages)

var typ_message = {} (thank you page messages)

8. Master Key Configuration

This chapter allows you to modify correctly the master key configuration of your landing page.

 

8.1       Master Key replacement

 

The Fiat Portal includes the Dynamic Master Key (DMK):

  1. MK are defined into the digitalData provided by the portal
  2. If the specific value from Fiat portal is empty, it is used the related MK value included into the hidden input of the form into the Landing Page. This MK must be replaced with the correct value given from the market.

In particular, the Master Key value that may be required to change are:

  • a) OFFER

<input type="hidden" id="field21" name="OFFER" value="">

  • b) CAMPAIGN

<input type="hidden" name="CAMPAIGN" value="5096">

  • c) MODEL DESCRIPTION

<input type="hidden" id="field18" name=" MODELDESCRIPTION" value="">

  • d) MODEL COMMERCIALCODE

<input type="hidden" id="field19" name="MODELCOMMERCIALCODE" value="">       

Remember that the following values are necessary for the Orchestrator system and must never change:

  • LEADSOURCECODE

<input type="hidden" id="field15" name="LEADSOURCECODE" value="MAT">

  • DTM_ENABLED

<input id="field92" name="DTM_ENABLED" type="hidden" value="1">

(e.g. for Fiat UK Standard TD)
