Introduction

The document has the purpose to illustrate the procedure in order to modify Alfaromeo forms.

  1. Assumption
    • LPID is the unique ID of the Landing Page
    • Each LP contains the form for a specific couple brand/market for different layouts, there will be different landing pages (e.g. Standard, Promo and Fleet/Business forms)
    • One LP contains only one CTA/form
  2. Alfaromeo Eloqua folder structur

    Alfaromeo Landing Pages are under All > EMEA > WEBPORTAL_INTEGRATION > ALFAROMEO path. For every country, there is a specific folder MARKET/LANGUAGE that contains other two subfolders with the corresponding pages:

    - CERT DMK (Certification LPID)

    - PROD DMK (Production LPID)

    If not present, create a TEST folder and duplicate the Production LP into it in order to apply the changes needed. If already present, copy always the content of the production LP into the test one to start to work from the most updated version available in production.

    When you create a new Landing Page you have to set the Microsite

    For Abarth it must be Alfa Romeo – Desktop.

    When done, report them in the corresponding PROD Landing Page.

    Remember that after modifying a Landing Page is always necessary an Alfaromeo AEM portal refresh.

    In some cases, when the changes affect:

    - Orchestrator values

    - CCDB values

    - adding and removing fields operation

    they must be validated by the ICT team through the internal process on the AEM Test environment, before to be reported in AEM Prod.

    This kind of changes must pass the ICT test before to be published in Production.
  3. Landing Page Permissions

    In order to integrate the Alfaromeo Landing page into AEM Portal it is necessary to give it the correct permissions. Follow this steps:

    1. Open Landing page from Eloqua platform
    2. In the right corner click on Actions button
    3. Click on Permissions
    4. Add group CloudConnector and give it View permission
    5. Save and save Landing page

LPID Identification

The LPID is the unique ID of the Landing Page. This chapter allows you to identify and locate the correct LPID of the form.

  1. LPID browser research
    To found quickly the LPID of the form to analyze, open the source code of the form in a browser (use a common developing tool).
    The LPID, in the HTML code, is identified with a specific (allows you to locate it faster):

    - tag: <input>

    - name: name ="LPID"

    <inputid ="lpid_td"name ="LPID"type ="hidden"value ="11248">(e.g. Alfaromeo UK TD)

  2. LPID Eloqua research
    To found the corresponding Landing Page in Eloqua, having the LPID, go to any LP and change the “id” parameter directly in query string:
    https://secure.p06.eloqua.com/Main.aspx#landing_pages&id={LPID}
    (e.g. for Alfaromeo UK TD: https://secure.p06.eloqua.com/Main.aspx#landing_pages&id=11248)

JS Identification

This chapter allows you to modify the correct JavaScript files included in the Landing Page that you are analyzing.

  1. JS LP research
    If you need to change a specific JavaScript file included in the page, you have to find the complete URL of the JS in the HTML code
    The name of the file to search will be the one after the following code autogenereted by Eloqua:

    /{…}_ or /NumericCode_

    The complete URL of the file will be the one corresponding to the src parameter of the script included in the page.

    				
    <script type="text/javascript" src="https://img06.en25.com/Web/FiatMarketing/{b380cbb4-3981-4916-99cb-734b11f85fc9}_20160520_Alfaromeo_validate_UK_TD.js?v=1"></script>
  2. JS File Storage research
    To search for the JS file in the File Storage section you can past the name of the file in the search bar and the result will be showed in the below section. Somethimes the results can be more then one, because in Eloqua is possible to upload files with the same name. In that case, you must compare the Actual URL of the file with the one included in the HTML page to locate the correct one to modify.

    Every time you need to modify a JavaScript file under Eloqua, open the Actual URL in the browser, save a copy on your local system and upload a new file under the same folder with the following naming convention:

    actualDate_fileName_versionNumber.js

  3. JS Files HTTPS Switch

    Eloqua for files under File Storage section has two different URLs, one to reach the resource in http and the other one to reach the resource in https:

    http://images.promo.fiat.com/Web/FiatMarketing/{b380cbb4-3981-4916-99cb-734b11f85fc9}_20160520_Alfaromeo_validate_UK_TD.js?v=1 (HTTP) https://img06.en25.com/Web/FiatMarketing/{b380cbb4-3981-4916-99cb-734b11f85fc9}_20160520_Alfaromeo_validate_UK_TD.js?v=1 (HTTPS)

    Due to the HTTPS switch of the AEM Alfaromeo portal, also the resources included in the portal pages must be accessed through the security protocol.

    When including a resource in the page is necessary to replace the domain of the Actual URL with the following one:

    http://images.promo.fiat.com

Label Configuration

This chapter allows you to modify correctly the message labels of your landing page.

  1. Label replacement

    If required to change a specific label, it is necessary to replace it directly into the HTML code.

  2. Thank you page message replacement
    If required to change a specific thank you page message, it is necessary to replace it directly in the HTML code.
    This is managed through an iframe in the same Landing Page.
    The typ message, in the HTML code, is identified with a specific (allows you to locate it faster):

    - tag : <p>

    - class : class="text--big"

  3. Error message replacement
    The validation of the page fields is managed through a JavaScript file, versioned under Eloqua in the “File Storage” section.
    The error message, in the HTML code, is identified with a specific (allows you to locate it faster):

    - tag : <span>

    - id : id="Error…"

    <spanid="ErrorNome_td"></span>

    In the specific JS validation file, included into the LP, there will be a function that manages the validation of the field. There you have to replace the error message with the new one.

    var str_error_nome_td = "Invalid name";

Privacy configuration

This chapter allows you to modify correctly the privacy section of your landing page.

  1. Privacy layout and labels

    Each market has different privacy configuration and different privacy messages. You can find the privacy section at the bottom of the page. Usually in this section of the HTML code there are specific div tags with the following classes:

    <divclass="form-row">
    <divclass="form-row__inner">

    This can help you to locate the specific section to edit. If required, edit the privacy of the forms whit the correct layout of the market or change the labels directly into the html.

  2. Privacy Disclaimer ID

    Each market has a different disclaimer code that corresponds to the document disclaimer number. The disclaimer’s id value can be modified changing its related value of the hidden input directly into the HTML code:

    a) DISCLAIMER

    <inputid="field29"name="DISCLAIMER"type="hidden" value="390"    (e.g. for Alfaromeo UK)

Master Key Configuration

This chapter allows you to modify correctly the master key configuration of your landing page.

  1. Master Key replacement
    The Alfaromeo Portal includes the Dynamic Master Key (DMK):

    a) MK are defined into the digitalData provided by the portal

    b) If the specific value from Alfaromeo portal is empty, it is used the related MK value included into the hidden input of the form into the Landing Page. This MK must be replaced with the correct value given from the market.

    In particular, the Master Key value that may be required to change are:

    a) OFFER <inputid ="field21_td"name ="OFFER"type ="hidden"value =" ">

    b) CAMPAIGN <inputid ="field27_td"name ="CAMPAIGN"type ="hidden"value =" ">

    c) MODEL DESCRIPTION <inputid ="field18_td"name ="MODELDESCRIPTION"type ="hidden"value =" ">

    d) MODEL COMMERCIALCODE <inputid ="field19_td"name ="MODELCOMMERCIALCODE"type ="hidden"value =" ">

    e) BUSINESS AREA <inputid ="field54_td"name ="BUSAREA"type ="hidden"value ="1">

    Remember that the following values are necessary for the Orchestrator system and must never change:

    - LEADSOURCECODE <inputid ="field15_td"name ="LEADSOURCECODE"type ="hidden"value ="MAT">

    - DTM_ENABLED <inputid ="field_dtm_enabled_td"name ="DTM_ENABLED"type ="hidden"value ="1">

    (e.g. for Alfaromeo TD UK)

Page Variables Configuration

This chapter allows you to modify correctly the variables configuration of your landing page.

  1. Variables configuration

    The Alfaromeo Landing Pages include some JS variables in the HTML page used by particular functions which are used to configure:

    • _offer
    • _campaing
    • brandCode
    • _offer_kmi
    • _campaign_kmi

    If asked to change that values,

    1. firstly you have to check if there is a script in the tag with the following declared var (allows you to locate it faster):

    								<eloquaform>
									…
									<script>
										   var _offer_kmi='3523';
										   var _campaign_kmi='7508';
										…
									</script>
									…
									</eloquaForm>                                                                                               
									//(e.f. for Alfaromeo IT KMI)

    2. Secondly tou have to change that values only in the above case, otherwise the problem is not in Eloqua scope.

    This kind of changes must pass the ICT test before to be published in Production.
  2. Digital Data configuration

    The Alfaromeo Landing Pages includes the digitalData configuration directly in the HTML page:

    • digitalData.page.component.form.leadType
    • digitalData.page.component.form.nameType
    • digitalData.page.component.form.stepName

    If asked to change that values,

    1. firstly, you have to check if there is a script in the tag with the following declared variables (allows you to locate it faster):

    							<eloquaform>
								…
								<script>
									   digitalData.page.component.form.stepName="form";
									…
								</script>
								…
								</eloquaForm>                                                                                               
								//(e.f. for Alfaromeo IT KMI)

    2. secondly, you have to change that values only in the above case, otherwise the problem is not in Eloqua scope.

    This kind of changes must pass the ICT test before to be published in Production.
