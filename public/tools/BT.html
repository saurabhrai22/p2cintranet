General BT request

  • Do not submit any BT request if changes are not live until it is promotion, wait for a day then next day we can submit the BT if changes are still not in live environment.
  • If there may be any potential escalation from market then submit the BT
  • Review the BT feedback, take necessary follow ups with LAL’s/Developers and close the BT on time
  • Check the issues internally with other teams, before raise any BT request
  • Don’t escalate any BT to Alessandro directly and inform LAL also, contact Sudhanshu for any super urgent issue, he may contact Alessandro
  • Make sure you are sending the report on specified time to Vijaya so that she can escalate BTs on time
  • Thursday is “ELOQUA Market Day” so we escalate all BTs on Wednesday once in a week

Eloqua BT request

  • Always check with your internal Eloqua SPOC and contact Ankit for ELOQUA BT escalation
  • Select only Eloqua project from project list in BT following are the priority level of ELOQUA:
  • 1-Block: if leads are invalid, or forms are invisible in live website, brochure is not downloading, forms are not submitting or something really critical,
  • 2-Urgent-2: incorrect validation of form in live, site any super urgent request from market
  • 3-High: Form validation changes, adding new business field in Auth or any issue which has greater priority than normal
  • 4- Normal: All request other than above
  • Summary: Marketxx_Brandxx_Form_Issue e.g. Belgium_Alfaromeo_Test drive form lead error
  • Description: try to make it more clear and always attach the necessary screenshots in document
