Form Testing - Step 1

  • Check if correct MK is provided.
  • Check model code is in the product catalogue. If it is in NO_PROD status then ask LAL for approval, without approval we can't proceed for ICT approval. Check if the model code is given as per correct market and brand.
  • We can check the updated model code on below location:
  • https://ts.accenture.com/sites/FGA_P2C_Proposal/SitePages/modelcode.aspx

 

Form Testing - Step 2

 

  • COMMON Validations needs to be followed for promotion form:

 

  • In Name and Last Name fields,
    • 2-50 total characters
    • any kind of alphabet letters
    • no consecutive spaces (keyboard locked)
    • no numbers or special characters
    • Special character exception:
  • - (hyphen) and ' (apostrophe) must be accepted and consecutive "--" and '' are not allowed.
  • In Email id fields,
    • Consecutive "@@" and ".." are not allowed. eg. test-test@@test...com.
    • Also, in the domain name, the characters should be more than 1 character. e.g. test@t.com should not be acceptable.
    • In the Confirmation email field, the email id should not be copied (copy & paste).
  • Check for Phone Number field also as per validation document.
  • These are the some common validation rule but always refer the "form fields CTA_ValidationRules" validation file at below location https://collab.extra.fcagroup.com/sites/lmdocumentation/Shared%20Documents/Forms/AllItems.aspx?RootFolder=%2Fsites%2Flmdocumentation%2FShared%20Documents%2FCRM%20guidelines%20-%20CTA%20fields%20and%20validation%20rules%2C%20MK%20setup%20procedures&FolderCTID=0x012000F1231E34AC4ED040AE5346E6441EBFF4& View=%7B5DF8B50B-3148-45BB-949E-3237F94104D9%7D  (Ask Kaushal/Drishti/Kirti for access to this location).

Validated highlighted fields.

Form Testing - Step 3

 

  • We need to check below details in F2:

 

  • Check all the data (basic information) is passing as we have filled in the form like title, name, surname, Address, email, confirm email, etc...
  • Check market, language and brand are passing correctly.
  • Below parameters to be checked by the provided MK are:
    • Offer ID
    • Campaign ID
    • CTA Type
    • Country Code
    • Brand
    • Market
    • Language
    • Level 1 & 4 should pass same as provided in MK File i.e. LEVEL1: web corporate & LEVEL4: brand respectively.
    • Level 2: For Promo form level 2 should pass promotions, i.e. LEVEL2: brand:market:promotions
    • Level 3: Level 3 should pass lead, i.e. LEVEL3: lead: type (e.g. for Test Drive form level 3 will be LEVEL3: lead:test-drive-request)
    • BUSAREA for business promo will pass "business" for other promo BUSAREA will pass blank or sales.

 

Form Testing – Step 3 continued

 

    • Check below Privacy values according to market:
      • PRIVACYPROFILATION
      • PRIVACYTHIRDPARTY
      • PRIVACYEXTRAEU
      • PRIVACYGEOLOCATION
      • MARKETING_POST1
      • MARKETING_MAIL1
      • MARKETING_SMS1
      • MARKETING_PHONE1

 

  • Check the visitor ID is passing or not.
  • If you find any issue while validation or in F2 data, then please check whether it is solve by ELOQUA Team or BT Team or we can change it through CPS/AEM(You). According to issue assign the task to respective team and once issue is resolved then again follow above procedure from step 2.
