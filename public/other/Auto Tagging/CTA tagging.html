CTA tagging

Generally every link that you can find on a page should contain a tagging attribute.

 

Some general rule for CTA tagging:

  • The attribute values are always in English language
  • The various components of attribute are separated by “:” -> “section1:section2:section3” (the sections can be only two, for instance in the CTA of page body)
  • The various section usually identify the components and subcomponents of the page:
    • section1: the section of the page. main-nav for header, footer for footer, cta for a CTA in page body (can be also top-cta and bottom-cta), content for home slider elements
    • section2: the menu or sub-section within the element is contained (e.g. shopping-tools)
  • If a value is composed of more than one word, the words will be separated by “-“ (e.g. “test-drive-request”)
  • The name of attribute can change for brand:
    • data-link_tracking=” ” for Fiat
    • link-tracking=” ” for Alfa Romeo Fiat Professional and Lancia
    • data-adobe=” ” for Jeep and Abarth
  • Basically, if you don’t know how to valorize a CTA, you can check how the same component has been valorized on other markets
  • Usually links that redirect to sub-sections of the model (design, interiors, security…) are not tagged
  • For Fiat, the model grid in the header is not tagged

 

Implementation:

Usually the CTA tagging can be done by data entry or by directly inserting the attribute in the HTML code of the page.

For Fiat, Fiat Professional, Alfa Romeo, Lancia and Abarth usually the CTA tagging is done by data entry. For Jeep at the moment all the CTAs are tagged by HTML attribute insertion.

Example of data entry:

  • Locate the element to tag in the various lists of the CMS
  • Open the element in edit mode
  • Populate the section dedicated to CTA tracking

Implementation on new AEM portals

In the new AEM portals, it’s not possible to directly insert the CTA tagging by data entry.

The tagging of CTAs is automatically created; the tag will be based on Analytics attributes of destination pages.

 

Common errors:

The most common reported errors are the following:

  • Missing tagging
  • Wrong tagging on CTA
  • Wrong tagging on homepage slider; many times, a new slide is inserted copying another slide and the tagging is not coherent with the content of the slide e.g.

 

The tagging is “slider:contact:quote-request:124-spider:spider” but the slide refers to qubo promotion page, so it is wrong and must be changed.

Checks of correctness:

  • Right click on the element to check > inspect element
  • Check the if data-adobe attribute is present
  • Check if the value of the attribute is coherent with the link (e.g. "cta:dealers" for the dealer locator)
