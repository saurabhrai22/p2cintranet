Datalayer and Pageload

The pageload is a sort of "tagging on the SEO":

The final scope of the pageload is showing information about the page within an environment variable "digitalData":

Here is some general guideline:

  • "category" and "pageInfo" usually should be populated for any page of the portal
  • In case of form presence on the page also the "form" sub-section under "component" have to be populated
  • In case of dealer locator presence on the page also the "dealerSearch" sub-section under "component" have to be populated
  • Usually the pageload is populated through data entry on SEO items, at the moment the only CMS where this data entry is not available is Teamsite for Jeep. In this case the pageload is realized by a javascript
  • Form informations on digitalData change any time a forms steps is completed (e.g. stepName="form", stepName="dealerselection", stepName="thankyou")

Implementation by data script:

(Jeep)

  • Under the folder /js/adobeTagging (if not available, create the folder), create two new files: adobeTracking.js and xLayer.js
  • In adobeTracking.js define the variable digitalData
  • In xLayer.js define the population of the digitalData variable based on the current page
  • Usually these two files are called in the file header_injection.html that is included in header.html, verify if the page recall them

Note that if the portal is tagged these two files will be available, in this case you have to edit xLayer.js if you want to tag the pageload of a new page.

For more specific information about pageload on Jeep, check the file Guidelines for Jeep Adobe tagging.docx.

Basically the pages have to be mapped in xLayer.js to valorize the digitalData values with the same values showed in data entry examples listed above for the other brands.

Common errors:

The most common reported errors are the following:

  • Wrong language or market in the digitalData variable (in this case the competence is of support@accenture.com for non-Jeep brands)
  • Missing population of digitalData variable
  • Wrong population of digitalData variable (e.g. pagetype as "homepage" on a promotion page)

Checks of correctness:

  • Digit "digitalData" in the browser console
  • Verify if the values inserted in page.category and page.pageInfo sections have coherence with the page (e.g. if you find "showroom" as page category in a promotion page).
  • If a form is present, verify the sub-component form under component section of digitalData
