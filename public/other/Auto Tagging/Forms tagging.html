Forms tagging

Basically the forms tagging is the population of “form” section under page.component section of digitalData variable, the valorization will change during the various step of the form (start, dealer selection, thank you



Usually the first step of form data is populated by data entry during the SEO tagging, the other steps will be populated by javascript.

This is an example of code for the thank you page if the request is submitted with success:



DTMtracker.track("permission", {
	  page: {
		   component: {
			  form: {
					   stepName: "thankyou",
					   nameType: "permission:kmi"
				  }
			  }
		  }
	  }, modelCode);

 

Implementation:

 

For the first step of the form usually is enough to populate the form section in the dataentry for SEO tagging.

If the form is contained in an iFrame (e.g. Fiat Professional), you have to include in the iFrame code the correct adobedtm and pagebottom scripts and the declaration of digitalData variable with the form fields populated.

 

e.g. test drive (form section):

"form" : {
			  "brochureDeliveryMethod" : "",
			  "contactMethod" : "",
			  "leadType" : "test-drive-request",
			  "models" : "",
			  "nameType" : "lead:test-drive-request",
			  "purchaseTimeframe" : "",
			  "stepName" : "form",
			  "transactionID" : ""
			  },

In the other forms step, when the form is submitted (POST), in case of success the DTMTracker.track function has to be called passing the other velues of form section (leadType, nameType, stepName and brochureDeliveryMethod in case of brochures that can be delivered in different ways).

Usually there are three types of stepName valorization:

  • form: at form start
  • dealerselection: if there is a step where you can select the dealer
  • thankyou: if the form is submitted correctly and the thank you page is visualized

 

Example of valorization when the thank you page is showed

 

For the valorization of leadType and nameType fields you can refer to form data population about the tagging of SEO (page 6).

leadType Adobe form Name

nameType Adobe form Name:Adobe Form Type

E.g. Quote Request:

 

 

leadType: quote-request

nameType: lead:quote-request

 

Common errors:

The most common reported errors are the following:

  • wrong leadType or nameType
  • missing “thankyou” valorization for stepName when the form is successfully submitted
  • missing “dealerselection” valorization for stepName ad dealer selection page visualization

 

Checks of correctness:

  • Digit “digitalData” in the browser console
  • Verify if the values inserted in page > component > form are correct in every step of the form
